# Task 1: Data Dictionary (Amazon Sales Report)

| Column Name | Data Type | Description | Business Relevance |
| :--- | :--- | :--- | :--- |
| `Order ID` | String | Unique identifier for each Amazon transaction | Crucial for tracking order volume and processing unique receipts. |
| `Date` | Datetime | The date the order was placed | Used for analyzing sales velocity, seasonality, and temporal trends. |
| `Status` | String | Current delivery/operational state of the order | Helps monitor processing bottlenecks, returns, or cancellations. |
| `Fulfilment` | String | Method used to ship items (AFN/MFN) | Identifies whether Amazon (FBA) or the Merchant (FBM) fulfilled the order. |
| `SKU` | String | Stock Keeping Unit identifier for inventory control | Essential for warehouse tracking and identifying top-selling product configurations. |
| `Category` | String | The broad classification of the item sold | Dictates high-level inventory distribution and marketing strategy focuses. |
| `Qty` | Integer | Total units of the product purchased per invoice | Vital for tracking inventory depletion rates and volume metrics. |
| `Amount` | Float | The price per unit of the item in local currency | Directly tracks product pricing structures and base operational revenue. |
| `Total_Sales_Value` | Float | Engineered Feature: `Qty` * `Amount` | Formulates the true gross revenue generated by individual line items. |
| `Order_Urgency` | String | Engineered Feature: Priority category of shipping level | Groups logistics metrics to evaluate courier speed demands. |
